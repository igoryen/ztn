<p><b>cURL</b> (/kÉ:(r)l/.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3"><span>[</span>3<span>]</span></a></sup>) is a computer software project providing a <a href="/wiki/Library_(computing)" title="Library (computing)">library</a> and <a href="/wiki/Command-line" title="Command-line" class="mw-redirect">command-line</a> tool for transferring data using various <a href="/wiki/Protocol_(computing)" title="Protocol (computing)" class="mw-redirect">protocols</a>. The cURL project produces two products, <b>libcurl</b> and <b>cURL</b>. It was first released in 1997.</p>

<h2><span class="mw-headline" id="libcurl">libcurl</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=CURL&amp;action=edit&amp;section=1" title="Edit section: libcurl">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>libcurl is a free client-side URL transfer <a href="/wiki/Library_(computing)" title="Library (computing)">library</a>, supporting <a href="/wiki/FTP" title="FTP" class="mw-redirect">FTP</a>, <a href="/wiki/FTPS" title="FTPS">FTPS</a>, <a href="/wiki/Gopher_(protocol)" title="Gopher (protocol)">Gopher</a>, <a href="/wiki/HTTP" title="HTTP" class="mw-redirect">HTTP</a>, <a href="/wiki/HTTPS" title="HTTPS">HTTPS</a>, <a href="/wiki/Secure_copy" title="Secure copy">SCP</a>, <a href="/wiki/SSH_File_Transfer_Protocol" title="SSH File Transfer Protocol">SFTP</a>, <a href="/wiki/Trivial_File_Transfer_Protocol" title="Trivial File Transfer Protocol">TFTP</a>, <a href="/wiki/Telnet" title="Telnet">Telnet</a>, <a href="/wiki/DICT" title="DICT">DICT</a>, the <a href="/wiki/File_URI_scheme" title="File URI scheme">file URI scheme</a>, <a href="/wiki/LDAP" title="LDAP" class="mw-redirect">LDAP</a>, LDAPS, <a href="/wiki/IMAP" title="IMAP" class="mw-redirect">IMAP</a>, <a href="/wiki/POP3" title="POP3" class="mw-redirect">POP3</a>, <a href="/wiki/SMTP" title="SMTP" class="mw-redirect">SMTP</a> and <a href="/wiki/RTSP" title="RTSP" class="mw-redirect">RTSP</a>. The library supports HTTPS certificates, <a href="/wiki/HTTP_POST" title="HTTP POST" class="mw-redirect">HTTP POST</a>, <a href="/wiki/HTTP_PUT" title="HTTP PUT" class="mw-redirect">HTTP PUT</a>, FTP uploading, <a href="/wiki/Kerberos_(protocol)" title="Kerberos (protocol)">Kerberos</a>, HTTP form-based upload, proxies, <a href="/wiki/Cookie_(web)" title="Cookie (web)" class="mw-redirect">cookies</a>, user-plus-password authentication, file transfer resume, and <a href="/wiki/HTTP_proxy" title="HTTP proxy" class="mw-redirect">HTTP proxy</a> tunneling.</p>
<p>The libcurl library is portable. It builds and works identically on several platforms, including <a href="/wiki/Sun_Solaris" title="Sun Solaris" class="mw-redirect">Solaris</a>, <a href="/wiki/NetBSD" title="NetBSD">NetBSD</a>, <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a>, <a href="/wiki/OpenBSD" title="OpenBSD">OpenBSD</a>, <a href="/wiki/Darwin_(operating_system)" title="Darwin (operating system)">Darwin</a>, <a href="/wiki/HPUX" title="HPUX" class="mw-redirect">HPUX</a>, <a href="/wiki/IRIX" title="IRIX">IRIX</a>, <a href="/wiki/AIX" title="AIX" class="mw-redirect">AIX</a>, <a href="/wiki/Tru64" title="Tru64" class="mw-redirect">Tru64</a>, <a href="/wiki/Linux" title="Linux">Linux</a>, <a href="/wiki/UnixWare" title="UnixWare">UnixWare</a>, <a href="/wiki/HURD" title="HURD" class="mw-redirect">HURD</a>, <a href="/wiki/Windows" title="Windows" class="mw-redirect">Windows</a>, <a href="/wiki/Symbian" title="Symbian">Symbian</a>, <a href="/wiki/AmigaOS" title="AmigaOS">AmigaOS</a>, <a href="/wiki/OS/2" title="OS/2">OS/2</a>, <a href="/wiki/BeOS" title="BeOS">BeOS</a>, <a href="/wiki/Mac_OS_X" title="Mac OS X" class="mw-redirect">Mac OS X</a>, <a href="/wiki/Apple_iOS" title="Apple iOS" class="mw-redirect">Apple iOS</a>, <a href="/wiki/Android_(operating_system)" title="Android (operating system)">Android</a>, <a href="/wiki/Ultrix" title="Ultrix">Ultrix</a>, <a href="/wiki/QNX_Neutrino" title="QNX Neutrino" class="mw-redirect">QNX Neutrino</a>, <a href="/wiki/BlackBerry_Tablet_OS" title="BlackBerry Tablet OS">BlackBerry Tablet OS</a> and <a href="/wiki/BlackBerry_10" title="BlackBerry 10">BlackBerry 10</a>,<sup id="cite_ref-4" class="reference"><a href="#cite_note-4"><span>[</span>4<span>]</span></a></sup> <a href="/wiki/OpenVMS" title="OpenVMS">OpenVMS</a>, <a href="/wiki/RISC_OS" title="RISC OS">RISC OS</a>, <a href="/wiki/Novell_NetWare" title="Novell NetWare" class="mw-redirect">Novell NetWare</a>, <a href="/wiki/DOS" title="DOS">DOS</a> and more.</p>
<p>The libcurl library is free, <a href="/wiki/Thread-safe" title="Thread-safe" class="mw-redirect">thread-safe</a>, <a href="/wiki/IPv6" title="IPv6">IPv6</a> compatible, and fast. <a href="/wiki/Language_binding" title="Language binding">Bindings</a> are available for more than 40 <a href="/wiki/Computer_languages" title="Computer languages" class="mw-redirect">languages</a>, including <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>/<a href="/wiki/C%2B%2B" title="C++">C++</a>, <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, <a href="/wiki/PHP" title="PHP">PHP</a> and <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>.</p>
<p>The libcurl library can support <a href="/wiki/Transport_Layer_Security" title="Transport Layer Security">SSL/TLS</a> through <a href="/wiki/OpenSSL" title="OpenSSL">OpenSSL</a>, <a href="/wiki/GnuTLS" title="GnuTLS">GnuTLS</a>, <a href="/wiki/Network_Security_Services" title="Network Security Services">NSS</a>, <a href="/wiki/CyaSSL" title="CyaSSL">CyaSSL</a>, <a href="/wiki/PolarSSL" title="PolarSSL" class="mw-redirect">PolarSSL</a>, <a rel="nofollow" class="external text" href="http://axtls.sourceforge.net/">axTLS</a>, <a href="/w/index.php?title=QSOSSL&amp;action=edit&amp;redlink=1" class="new" title="QSOSSL (page does not exist)">QSOSSL</a> on <a href="/wiki/IBM_i" title="IBM i">IBM i</a>, <a href="/wiki/Secure_Transport" title="Secure Transport" class="mw-redirect">Secure Transport</a> on OS X and iOS, and <a href="/wiki/Security_Support_Provider_Interface" title="Security Support Provider Interface">SChannel</a> on Windows.</p>
<h2><span class="mw-headline" id="cURL">cURL</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=CURL&amp;action=edit&amp;section=2" title="Edit section: cURL">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A command line tool for getting or sending files using <a href="/wiki/URL" title="URL" class="mw-redirect">URL</a> syntax.</p>
<p>Since cURL uses libcurl, it supports a range of common Internet protocols, currently including HTTP, HTTPS, FTP, FTPS, SCP, SFTP, TFTP, LDAP, LDAPS, DICT, TELNET, FILE, IMAP, POP3, SMTP and RTSP (the last four only in versions newer than 7.20.0 or 9 February 2010).</p>
<h2><span class="mw-headline" id="Examples_of_cURL_use_from_command_line">Examples of cURL use from command line</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=CURL&amp;action=edit&amp;section=3" title="Edit section: Examples of cURL use from command line">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Basic use of cURL involves simply typing curl at the command line, followed by the URL of the output to retrieve.</p>
<p>To retrieve the <a href="/wiki/Example.com" title="Example.com">example.com</a> homepage, type:</p>
<pre>
curl www.example.com
</pre>
<p>cURL defaults to displaying the output it retrieves to the standard output specified on the system (usually the terminal window). So running the command above would, on most systems, display the www.example.com source-code in the terminal window.</p>
<p>cURL can write the output it retrieves to a file with the -o flag, thus:</p>
<pre>
curl -o example.html www.example.com
</pre>
<p>This will store the source code for www.example.com into a file named example.html. While retrieving output, cURL will display a progress-bar showing how much of the output has downloaded. Note however that cURL does not show a progress bar when preparing to display the output in the terminal window, since a progress-bar is likely to interfere with the display of the output.</p>
<p>To download output to a file that has the same name as on the system it originates from, use the -O flag, for example:</p>
<pre>
curl -O www.example.com/example.html
</pre>
<p>If the server responds that the file (example.html) is moved to a different location (indicated with a Location: header and a 3XX response code), use the -L flag, for example:</p>
<pre>
curl -OL www.example.com/example.html
</pre>
<p>Curl offers many other features such as proxy support, user authentication, FTP upload, HTTP post, SSL connections, cookies, file transfer resume, Metalink, as well as various other features.</p>
